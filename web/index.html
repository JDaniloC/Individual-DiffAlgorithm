Explicação

O algorítmo realiza o Longest Common Sequence comparando os dois arquivos (linha por linha), no qual vai devolver o tamanho da sequência mais longa. Mas como a solução é obtida usando programação dinâmica através de uma matriz, então é possível ir voltando na matriz para se obter esse termo mais longo.


Com essa informação, é comparado com o arquivo original, no qual não haverá cores caso a linha estiver intácta. Caso tiver alguma linha que não está no arquivo original, seja antes ou depois, então será considerado como adição(+), caso alguma linha não aparecer no arquivo modificado então será considerado como remoção(-). E por fim, toda linha intácta que tiver sido movida devido alguma adição(+) ou remoção(-), será considerada como modificação, sendo assim colorida de azul.


O backend do frontend foi feito com HTML/CSS/JS puro, já o backend foi feito em Python, sendo conectado usando Eel já que o algorítmo de LCS foi feito em Python. Dessa forma, os arquivos são capturados via javascript, e ao pressionar o botão de computar, então ele irá enviar o texto de ambos os arquivos para o script python, que por sua vez devolve um novo texto com os sinais (+, -, M) para o Javascript, que por fim envia para o HTML renderizar.
